# 货代系统异常预警规则配置器 - Figma UI 设计提示词

## 一、 全局页面布局规范 (Global Layout)
- **页面形态**：右侧滑出抽屉 (Right Drawer) 或 全屏沉浸式表单页 (Full-page Modal)。
- **视觉层级**：分为三大区块：【基础信息】、【触发条件 (IF) 逻辑组】、【执行动作 (THEN)】。区块间需有明显的视觉分割（卡片式或浅灰色底色+分割线）。

---

## 二、 模块组件拆解 (Component Details)

### 区块 1：基础信息 (Basic Information)
- **规则名称**：`Input (必填)`，占位符：“例如：美线 ETA 严重延误预警”。
- **适用业务域**：`Select (多选胶囊/Tag)`，选项：`海运整箱(FCL)`、`海运拼箱(LCL)`、`空运(AIR)`、`全局(Global)`。
- **防打扰冷却期**：`Input Number` + `单位(小时)`。需附带 `Tooltip` 图标提示：“触发报警后，同一票单据在此时间内不会重复报警”。
- **规则状态**：`Toggle Switch`（启用/停用）。

### 区块 2：核心条件构造器 (Condition Builder - IF)
*交互核心：呈现出明显的“拼积木”感，后一个控件的显隐/选项受前一个控件控制。*

- **外层容器**：带虚线边框或浅色背景的“逻辑组卡片”。
- **顶部关系选择**：`Radio` 或 `Select` -> `[满足以下全部条件 (AND)]` / `[满足以下任意条件 (OR)]`。
- **动态条件行 (Condition Row)**：每行包含 4 个动态槽位 (Slots)，右侧常驻 `[🗑️ 删除行]` 幽灵图标按钮。
- **底部操作**：`[+ 添加条件]` 主按钮。

**【条件行槽位联动逻辑表】**（设计师需根据此表绘制不同状态的拼接组合）：
| 槽位 1：监控对象 (树状 Select) | 槽位 2：操作符 (动态 Select) | 槽位 3：输入阈值/内容 | 槽位 4：单位 |
| :--- | :--- | :--- | :--- |
| **日期类** (最新 ETD / 最新 ATD / 最新 ETA) | `对比基准延误大于` / `对比基准提前大于` | `Input Number` | `Select` (小时/天) |
| **属性类** (最新船名 / 最新航次) | `不等于基准数据` / `等于基准数据` | *(UI 隐藏)* | *(UI 隐藏)* |
| **路由类** (最新中转港列表 / 最新目的港) | `发生变更(对比基准)` | *(UI 隐藏)* | *(UI 隐藏)* |
| **停滞类** (当前轨迹节点) | `停留时长大于` | `Input Number` | `Select` (小时/天) |
| **事件类** (最新轨迹事件) | `包含高危代码` / `不包含` | `Select(多选标签)` | *(UI 隐藏)* |

### 区块 3：执行动作区 (Action Execution - THEN)
- **动作类型**：`Select`，默认选项：`发送通知`。
- **通知模板选择**：`Select`，下拉选择已配置的模板名称。
- **接收人配置**：`Tag Input / Multi-Select`。支持选择动态角色（`@责任操作(OP)`、`@责任销售(Sales)`）或直接输入邮箱地址生成 Tag。
- **通知渠道**：`Checkbox Group` -> `[x] 站内信`、`[x] 邮件`、`[ ] 企业微信/钉钉`。

## 三、 必须出图的 4 个高保真拼接态 (Mockup Prompts)
*设计稿中必须直观展示以下 4 种业务场景的条件行 UI 拼接效果，用于需求评审演示：*

1. **场景一（日期类比对 - 需录入数值）**：
   `[最新 ETA] ▾`  `[对比基准延误大于] ▾`  `[ 72 ]`  `[ 小时 ] ▾` 

2. **场景二（属性类比对 - 隐藏后续输入框）**：
   `[最新船名] ▾`  `[不等于基准数据] ▾`
   *(需在旁边加上灰色小字或 Tooltip：系统将自动与单票首次获取的基准船名进行比对)*

3. **场景三（路由类比对 - 隐藏后续输入框）**：
   `[最新中转港列表] ▾`  `[发生变更(对比基准)] ▾`

4. **场景四（事件类多选 - 扩展场景展示）**：
   `[最新轨迹事件] ▾`  `[包含高危代码] ▾`  `[ 🛡️ Customs Hold 查验 ✖ ] [ 💥 Damaged 破损 ✖ ]`

## 四、 附加 UX 交互提示 (UX Tips)
- **模拟测试入口**：在底部 `[取消] / [保存]` 按钮组旁边，增加一个幽灵按钮 `[▶ 模拟测试 (Test Run)]`。
- **前置警告条**：在页面顶部放置一个 `Warning Alert` 横幅：“注意：使用对比基准规则前，需确保系统内已成功获取第一笔基准数据，否则规则将被跳过。”
